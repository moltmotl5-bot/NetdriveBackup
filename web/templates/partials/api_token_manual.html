REST API 使用手冊

請使用本頁建立的 API Token 呼叫外部整合介面。

驗證方式

  • 路徑前綴：/api/v1
  • 每次請求標頭 X-API-Key = 建立當下顯示的完整 nccm_… token
  • 權限：inventory:read（庫存列表）
  • 尚無啟用中且未過期 token 時，受保護的 API 無法使用
  • 建立時必填有效天數（預設 90、最長 365）；過期回 401 API token expired
  • 無 expires_at 的舊 token 視為已過期，請停用並重建

端點

方法路徑驗證說明
GET /api/v1/health 無 服務狀態
GET /api/v1/inventory Token 庫存列表（JSON）

GET /api/v1/inventory 查詢參數

參數說明
site依站點篩選
vendor依廠牌篩選
q關鍵字（IP、hostname、型號、序號）
limit筆數上限，預設 100，最大 500
offset略過筆數，預設 0

回應欄位（每筆設備）

站點、IP、埠、主機名稱、廠牌、版本、型號摘要、序號摘要、備份次數，以及 Stack／HA 相關欄位（見 JSON 回應）。

HTTP 狀態碼

  • 200 — 成功
  • 401 — 缺少、無效或已過期的 X-API-Key
  • 403 — 權限不足
  • 500 — 尚無任何未過期且啟用中的 API Token

curl 範例

export NCCM_API_TOKEN="nccm_your_token_from_this_page"

curl -sS -H "X-API-Key: $NCCM_API_TOKEN" \
  "http://localhost:8501/api/v1/inventory?limit=20"

curl -sS "http://localhost:8501/api/v1/health"

Postman

  1. Method：GET，URL：{{baseUrl}}/api/v1/inventory
  2. Headers：X-API-Key = 完整 token（勿加 Bearer）
  3. 無需 Cookie／Session

安全建議

  • Token 僅建立時顯示一次；遺失請停用舊 token 並建立新 token
  • 定期輪替；不同整合請使用不同名稱的 token
  • 審計寫入 store/audit/audit.db；admin 可於「審計日誌」查看／匯出 CSV
  • 生產環境請以 HTTPS 存取，避免 token 明文傳輸
