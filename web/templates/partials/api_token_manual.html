REST API 使用手冊

外部整合請使用本頁建立的 API Token（存於 portal_auth.db，僅雜湊）。不在 .env 設定 API 金鑰。

驗證方式

  • 路徑前綴：/api/v1（不經 Web Session 閘道）
  • 每次請求標頭 X-API-Key = 建立當下顯示的完整 nccm_… token
  • Scope：inventory:read（目前庫存列表端點）
  • 伺服器尚無任何啟用中 token 時，受保護端點回 500（fail-closed）

端點

方法路徑驗證說明
GET /api/v1/health 無 回 {"status":"ok"}
GET /api/v1/inventory Token 庫存 JSON 陣列（Stack/HA 已展開）

GET /api/v1/inventory 查詢參數

參數說明
site依站點篩選
vendor依廠牌篩選
q關鍵字（IP、hostname、model、serial）
limit筆數上限，預設 100，最大 500
offset略過筆數，預設 0

回應欄位（每筆設備）

device_id、site、ip、port、hostname、vendor、sw_version、model_summary、serial_summary、snapshot_count、stack_switch、stack_role、is_config_anchor、cluster_type

HTTP 狀態碼

  • 200 — 成功
  • 401 — 缺少或無效的 X-API-Key
  • 403 — Token scope 不足
  • 500 — 伺服器尚未建立任何啟用中的 API Token

curl 範例

export NCCM_API_TOKEN="nccm_your_token_from_this_page"

curl -sS -H "X-API-Key: $NCCM_API_TOKEN" \
  "http://localhost:8501/api/v1/inventory?limit=20"

curl -sS "http://localhost:8501/api/v1/health"

Postman

  1. Method：GET，URL：{{baseUrl}}/api/v1/inventory
  2. Headers：X-API-Key = 完整 token（勿加 Bearer）
  3. 無需 Cookie／Session

安全與維運

  • Token 僅建立時顯示一次；遺失請停用舊 token 並建立新 token
  • 定期輪替；不同整合（Postman、腳本、第三方）請用不同名稱的 token
  • 生產環境請以 HTTPS 反向代理，避免 token 明文傳輸
  • 備份 store/portal_auth.db 時一併保護 token 雜湊

進階：可於部署環境設定選用變數 NCCM_API_TOKEN_PEPPER 加強雜湊（勿遺失，否則既有 token 失效）。此項不寫入 .env.example。
