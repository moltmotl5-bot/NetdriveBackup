NetdriverBackup NCCM v3 使用手冊

依 Portal 側欄功能分類的操作說明。登入後可從側欄「使用手冊」或網址 /help 開啟本頁。

產品簡介

NCCM v3 以 Web Portal 搭配 NetDriver Agent，透過 SSH 擷取網路設備組態與狀態，寫入本機 store/ 目錄與 SQLite 索引。

瀏覽器 → nccm-portal (FastAPI) → netdriver-agent (SSH) → 設備
                ↓
            store/（快照 + index.db）
元件預設容器職責
Portalnccm-portal登入、備份 UI、庫存、排程、API
NetDriver Agentnccm-netdriver-agentSSH 連線、廠牌 plugin、探測
資料主機 ./store快照與索引（請持久化備份）

支援廠牌：Cisco（IOS / NX-OS）、Huawei CE、Fortinet FortiGate。不支援 WLC。

安裝與啟動

在專案根目錄（需 Docker 24+ 與 Compose v2）：

git clone https://github.com/moltmotl5-bot/NetdriveBackup.git
cd NetdriveBackup
cp .env.example .env
# 編輯 NCCM_ADMIN_PASS（至少 12 字元）
chmod 600 .env
mkdir -p store
docker compose up -d --build
URL用途
http://localhost:8501/loginPortal 登入
http://localhost:8501/help本手冊
http://localhost:8000/docsAgent OpenAPI（除錯）

檢查：

docker compose ps
docker compose logs -f portal
docker compose logs -f netdriver-agent
curl -s http://localhost:8501/health

側欄 Agent 顯示 Online（綠）表示 Portal 可連 Agent。Offline（紅）請見「疑難排解」。

Agent 容器須能從 Docker 網路 SSH 至設備管理 IP:Port。若設備僅主機直連可達，需調整 Agent 網路模式（見 README 進階說明）。

登入與角色

首次啟動以 .env 的 NCCM_ADMIN_USER／NCCM_ADMIN_PASS 登入後，系統會強制變更密碼。之後日常請用 admin 建立的帳號；僅改 .env 不會更新已存在使用者的密碼。

角色側欄可見權限摘要
admin 全部 + 使用者／API Token／審計 完整操作與管理
operator 備份、總表、鄰居、介面、排程、手冊 備份、排程、索引、retention、config 下載；無使用者管理
viewer 總表、鄰居、介面、排程（唯讀）、手冊 不可批次備份、不可變更排程、不可下載 config
能力adminoperatorviewer
批次備份／重建索引／快照保留✓✓✗
排程（建立／執行／編輯）✓✓✗
排程（唯讀）／總表／鄰居／介面✓✓✓
config 下載✓✓✗
使用者／API Token／審計✓✗✗

批次備份

側欄「批次備份」→ /backup。單次對 CSV 清單執行真實備份。

  1. 上傳 CSV 或貼上內容（格式見「設備 CSV」）。
  2. 輸入 SSH 帳號、密碼；Huawei／Fortinet 視需要填 Enable 密碼；Cisco 備份為 login 模式、通常不需 enable。
  3. 按開始；背景 job 以 SSE 即時顯示每台 log 與結果表。

廠牌備份差異（摘要）

廠牌Agent 模式設定備份（摘要）
CiscologinNexus：show running-config；IOS：show running-config view full
Huaweienabledisplay current-configuration 等；stack／序號相關 CLI 一併擷取
Fortinetenableshow full-configuration 等

成功後寫入 store/<Site>/<IP>__<Hostname>/snapshots/<UTC>/，並更新 index.db。失敗時請看 SSE 中的 [config] 與 CLI tail。

設備總表與版控

側欄「設備總表與版控」→ /inventory。

  • 列出已索引設備；Cisco Stack、Huawei iStack、FortiGate HA 展開為每台實體一行（含 Stack#、Role、序號）。
  • 選單一設備可檢視快照版本列表、各版 artifact。
  • 選兩個版本可檢視 Config Diff（文字 diff）。
  • admin／operator：下載 config、重建索引（掃描整個 store）、快照保留（先 dry-run 預覽刪除筆數，再執行清理）。
Stack／HA 展開異常時：對設備重新備份 →「重建索引」→ 若程式已更新請 docker compose up -d --build portal。

CDP/LLDP 鄰居

側欄「CDP/LLDP 鄰居」→ /neighbors。由最新快照解析鄰居表（Cisco CDP、Huawei／其他 LLDP）。

  • 上方設備列表與總表相同展開邏輯（Stack／HA）。
  • 選設備後下方顯示鄰居列：本端介面、鄰居名稱、鄰居 IP／介面等（依快照內容）。
  • 支援搜尋過濾。

Interface Map

側欄「Interface Map」→ /interfaces。合併 config.txt 介面定義與 interfaces.txt 狀態。

  • 設備列表含 Stack#／Role（與總表一致）。
  • 選設備後顯示埠位表：介面名稱、描述、admin/oper 狀態、IP 等（依廠牌與快照而異）。

排程備份

側欄「排程備份」→ /schedules。以日為週期自動備份（與批次備份共用 runner）。

建立排程（admin／operator）

  1. 填名稱、間隔（日）、SSH 帳密（Enable 可選）。
  2. 上傳 CSV 檔案（不支援貼文字）→「上傳並連線測試」。
  3. Agent TCP 探測每台；預覽頁顯示 OK／FAIL。
  4. 「確認建立排程」→ 僅 OK 設備納入。

列表操作

按鈕說明
設備 (N)彈窗顯示納入清單（viewer 可檢視）
立即備份立刻啟動非同步 job
編輯名稱、間隔、帳密；或重新上傳 CSV（需再探測確認）
啟用／停用停用後背景 watcher 不自動觸發
刪除移除排程（需確認）

執行歷史

頁面下方列出最近執行：狀態（running／done／failed／skipped）、觸發來源（manual／watcher）、成功台數與摘要。所有角色可檢視。

背景 watcher 約每 60 秒掃描到期排程；同一時間僅允許一個排程 job（避免重複備份）。

使用者管理

僅 admin。側欄「使用者管理」→ /admin/users。

  • 建立使用者（帳號、密碼 ≥12 字元、角色 admin／operator／viewer）。
  • 重設密碼、變更角色、刪除使用者。

API Token

僅 admin。側欄「API Token」→ /admin/api-tokens。建立 REST 整合用 token（詳見「REST API」）。

  • 建立時必填有效天數（預設 90、最長 365）；明文 token 僅顯示一次。
  • 可停用／刪除 token；過期 token 呼叫 API 回 401。

審計日誌

僅 admin。側欄「審計日誌」→ /admin/audit。

  • 記錄登入、備份、排程、API 請求、使用者變更等事件。
  • 可依事件類型篩選、匯出 CSV。

REST API

路徑前綴 /api/v1。與 Web Session 分開：不需 Cookie，使用 X-API-Key 標頭。

取得 Token

  1. 以 admin 登入 →「API Token」→ 建立 token。
  2. 複製完整 nccm_… 字串（僅建立當下顯示）。

端點

方法路徑驗證說明
GET/api/v1/health無{"status":"ok"}
GET/api/v1/inventoryToken庫存 JSON 列表（scope inventory:read）

GET /api/v1/inventory 查詢參數

參數說明
site站點篩選
vendor廠牌篩選
q關鍵字（IP、hostname、型號、序號）
limit筆數上限，預設 100，最大 500
offset略過筆數，預設 0

回應欄位（每筆）

device_id、site、ip、port、hostname、vendor、sw_version、model_summary、serial_summary、snapshot_count、stack_switch、stack_role、is_config_anchor、cluster_type 等。

HTTP 狀態碼

  • 200 — 成功
  • 401 — 缺少、無效或已過期 API Key
  • 403 — scope 不足
  • 500 — 伺服器尚無任何啟用且未過期的 API Token

curl 範例

export NCCM_API_TOKEN="nccm_your_token_from_portal"

# 健康檢查（無需 token）
curl -sS "http://localhost:8501/api/v1/health"

# 庫存列表
curl -sS -H "X-API-Key: $NCCM_API_TOKEN" \
  "http://localhost:8501/api/v1/inventory?limit=20&site=hq"

# 關鍵字搜尋
curl -sS -H "X-API-Key: $NCCM_API_TOKEN" \
  "http://localhost:8501/api/v1/inventory?q=10.1.1&vendor=cisco"

Postman

  1. Method GET，URL http://localhost:8501/api/v1/inventory
  2. Header X-API-Key = 完整 token（勿用 Bearer）
  3. 無需 Cookie
生產環境請以 HTTPS 反向代理；token 遺失請停用舊 token 並建立新 token。

下載 config 檔、備份歷史等尚未提供 REST 端點；請用 Web「設備總表」或批次備份。

Docker 更新

任何更新前請先備份主機 ./store 目錄。

日常更新（程式小改）

git pull
docker compose build --no-cache
docker compose up -d

完整更新（升級版本或行為異常）

docker compose down
git pull
docker compose build --no-cache
docker compose up -d --build

Agent unhealthy／volume 權限問題

docker compose down -v
git pull
docker compose build --no-cache
docker compose up -d --build

-v 會刪除 Compose volumes；不會刪除 bind mount 的 ./store，但仍請確認 store 已備份。

僅更新單一服務

# 僅 Agent
docker compose build --no-cache netdriver-agent
docker compose up -d netdriver-agent

# 僅 Portal
docker compose build portal
docker compose up -d portal

更新後檢查

docker compose ps
docker compose logs -f netdriver-agent
curl -s http://localhost:8501/health

若索引結構或 hostname 解析有變，請於 Web「設備總表」按 重建索引。

疑難排解

現象建議
側欄 Agent Offlinedocker compose ps；docker compose logs netdriver-agent；curl http://localhost:8000/health
備份全失敗Agent 容器能否 SSH 至 CSV 的 IP:Port；帳密正確；看 SSE log
Huawei Unsupported mode: login請更新 Portal 映像（Huawei 需 enable 模式 payload）
庫存缺設備／只有一台同 Site+IP 需不同 port 或 hostname；按「重建索引」
Stack／HA 展開不對重新備份 → 重建索引 → rebuild portal
排程一直 running等 job 完成；更新至最新 main；必要時停用再啟用排程
Agent unhealthydocker compose down -v 後重建（見 Docker 更新）
API 401token 過期或打錯；至 API Token 頁重建
API 500尚無啟用中的 API Token
