NetdriverBackup NCCM v3 使用手冊

企業網路設備組態備份、庫存與埠位檢視 — 安裝、設定與日常操作完整說明。
原始碼：github.com/moltmotl5-bot/NetdriveBackup

產品簡介

NCCM v3（Network Configuration Change Management）以 FastAPI Web 入口搭配 NetDriver Agent，透過 SSH 對網路設備執行 CLI，擷取組態與狀態並寫入本機 store/ 目錄與 SQLite 索引。

  • 支援廠牌：Cisco（IOS / NX-OS）、Fortinet FortiGate、Huawei CE（v3 路線圖不含 WLC）。
  • Web 功能：批次備份、設備總表與版控（含 Config Diff／快照保留）、CDP/LLDP 鄰居、Interface Map、排程備份（CSV 上傳、Agent 探測、週期真實備份）。角色：admin／operator／viewer（viewer 對排程為唯讀）。
  • 正式部署：專案根目錄 docker-compose.yml（Portal + Agent）。舊版 Streamlit 已移至 legacy/。

安裝 — Docker Compose（建議）

  1. 取得專案原始碼並進入目錄： 
    git clone https://github.com/moltmotl5-bot/NetdriveBackup.git
cd NetdriveBackup
  2. 建立 Portal 登入設定： 
    cp .env.example .env
# 編輯 .env：設定 NCCM_ADMIN_USER、NCCM_ADMIN_PASS（至少 12 字元，不可為範例弱密碼）
chmod 600 .env
  3. 建立備份目錄並啟動服務： 
    mkdir -p store
docker compose up -d --build
    首次建置會編譯 netdrive-agent/ 與 Portal 映像，需能存取基礎映像 registry（含 ghcr.io）。
  4. 若 Agent 狀態為 unhealthy（常見於舊版 agent-logs volume）：docker compose down -v 後再 docker compose up -d --build。
  5. 開啟 Web：http://localhost:8501/login（埠可經 NCCM_PORT 調整）。
URL用途
http://<host>:8501/loginNCCM Web（需登入）
http://<host>:8501/healthPortal 健康檢查（JSON，含 Agent 是否可連）
http://<host>:8000/docsNetDriver OpenAPI（除錯；生產環境建議限制存取）
http://<host>:8000/healthAgent 健康檢查
停止服務：docker compose down。僅重啟 Portal：docker compose restart portal。

設定 — REST API（API Token）

路徑前綴 /api/v1。與 Web Session 登入分開：admin 於 Portal「API Token」建立 token（存於 portal_auth.db，僅雜湊）；建立時必填有效天數（預設 90、最長 365）；過期回 401。請求帶 X-API-Key。審計寫入 store/audit/audit.db，admin「審計日誌」可篩選／匯出 CSV。完整端點見 API Token 頁 REST 手冊與 docs/inventory-api-contract.md。

  • GET /api/v1/health — 無需金鑰。
  • GET /api/v1/inventory — 需未過期 token（scope inventory:read）；尚無可用 token 時回 500（fail-closed）。
curl -sS -H "X-API-Key: YOUR_TOKEN" "http://localhost:8501/api/v1/inventory?limit=20"

完整說明見專案 README.md「API 介面」章節與 docs/inventory-api-contract.md 欄位契約。

當設備管理網僅主機可達時，編輯 docker-compose.yml 的 netdriver-agent：

使用 — 登入與側欄

  1. 開啟 /login，輸入 .env 的 NCCM_ADMIN_USER / NCCM_ADMIN_PASS。
  2. 登入成功後導向「批次備份」；左側導覽四頁：批次備份、設備總表與版控、CDP/LLDP 鄰居、Interface Map。
  3. 側欄顯示 Agent 狀態為 Online 或 Offline（不顯示 URL／埠）。
  4. /logout 可登出。

使用 — 批次備份

  1. 準備 CSV（上傳檔案或貼上文字）。
  2. 輸入設備 SSH 使用者名稱、密碼。
  3. 支援廠牌（CSV Vendor 小寫）：Cisco（IOS / IOS-XE Catalyst、Stack；Nexus）、Fortinet（FortiGate HA）、Huawei（VRP）。不支援 WLC。多廠牌帳密不同時請分次備份。
  4. 按開始備份；工作於背景執行，頁面以 SSE 串流顯示每台設備進度與錯誤。
  5. 成功後快照寫入 store/ 並更新 index.db。

每台設備大致流程

  • 依 Vendor 連線 NetDriver（必要時先以 version 指令自動辨識 Model/Version 後重連）。
  • 擷取 running config、version、CDP/LLDP（依廠牌）、interfaces 等。
  • FortiGate 使用 show full-configuration；Huawei 使用 display current-configuration 等。
同一表單一組 SSH 帳密套用到 CSV 全部列。若各廠牌帳密不同，請分次備份或拆成多份 CSV。

使用 — 設備總表與版控

  • 列表來自 index.db，可依 Site、Vendor（下拉，選項為已備份設備之廠牌）、關鍵字（IP / Hostname / Model / SN）篩選；鄰居與 Interface Map 頁同樣使用 Vendor 下拉。
  • 設備表約顯示 10 列高度，其餘於表內捲動；點列或「檢視」開啟下方詳情。
  • 詳情含型號、OS 版本、序號、快照時間列表；可預覽該次 config.txt（Running-Config），並以下載取得完整檔案（GET /inventory/download/config，需登入）。下載旁有警示：檔案可能無法直接用於還原，套用前請自行驗證。
  • Hostname 優先從 running-config 擷取；若舊快照顯示 unknown，可按「重建索引」從 config.txt 回填。
  • Cisco Stack / Huawei iStack / FortiGate HA：總表、鄰居、Interface Map 的設備列表皆每台實體機一行（Stack#、Role、各自 Model/Serial）。 Stack 角色與 Huawei 對齊：Cisco／Huawei 皆顯示 Master / Standby / Member（Cisco CLI 的 Active→Master；勿與 Forti HA 的 Primary/Secondary 混淆）。 FortiGate HA 仍為 Primary / Secondary。 Cisco／Huawei 成員 hostname 為 管理名 · SW#；Forti 為 HA 解析之主機名。錨點列標 · 組態；虛擬 IP 只備一份 config。Cisco 備份含 show switch（stack_info.txt，非 Nexus）。Huawei 備份含 display stack → stack_info.txt（失敗 soft-skip）；僅有 stack CLI 且 ≥2 成員才展開，manufacture 只補序號。異常時重新備份後按「重建索引」；更新 Portal 映像後請 docker compose up -d --build portal。
  • 重建索引：掃描整個 store/ 重建 index.db（手動還原資料或修復索引時使用）。與下方「快照保留」不同：重建不刪快照目錄。
  • Config Diff：於詳情選兩個快照版本，比對 config.txt 的 unified diff（僅檢視，不改檔）。
  • 快照空間管理見下一節「快照保留」。

使用 — 快照保留（預覽刪除／執行清理）

設備每次成功備份會在 store/<site>/<ip>_<port>/snapshots/<UTC時間>/ 新增一層快照，並寫入 index.db。長期累積會佔磁碟；快照保留（retention）用來按設備只留下最近 N 份成功快照，刪除更舊者。

權限：僅 admin／operator 可見按鈕（viewer 無）。操作會寫入審計 store/audit/audit.db（事件 snapshot_retention）。

保留最近 N 份

  • 數字框 保留最近 N 份（預設 10，範圍 1–500）。
  • 演算法：依 device_id 分組，只看 status = ok 的快照，按 created_at 由新到舊排序；保留前 N 筆，第 N+1 筆起列為待刪。
  • 某設備總快照數 ≤ N 時，該設備不會刪任何東西。
  • N 下限為 1（實作會 max(1, keep_last)），避免一次清光。

作用範圍（是否選中設備）

  • 若總表已選中某一設備（詳情開啟，表單隱藏欄 device_id 有值）：只對該台算保留／刪除。
  • 若未選設備（device_id 空）：對全部設備各自套用「保留最近 N 份」。

「預覽刪除（dry-run）」

  • 只計算、不刪除：不 rmtree、不改 index.db。
  • 送出後頁面上方橫幅類似：快照保留：dry — keep_last=10，筆數=K，其中 K = 將會刪除的快照筆數。
  • 用途：先確認「若現在清理會動到幾份」再決定是否執行。建議每次先預覽、後清理。

「執行清理」

  • 瀏覽器會先 confirm 確認。
  • 真的刪除 dry-run 同一套候選清單：
    1. 刪除對應快照目錄（shutil.rmtree）。
    2. 自 index.db 的 snapshots 表刪除該列。
    3. 重算該設備的 snapshot_count、最新快照欄位；若已無成功快照則移除 devices／相關 stack_units 列。
  • 完成後橫幅：快照保留：done — keep_last=…，筆數=實際刪除目錄數。
  • 目錄不存在或刪除失敗會列入 skipped（不中斷整批）；仍會盡量清 DB 候選 id。

與「重建索引」的差異

操作磁碟快照目錄index.db用途
重建索引 不動 依 store/ 全掃重建 修索引／回填 hostname 等
預覽刪除 不動 不動 看會刪幾份
執行清理 刪舊快照目錄 刪 snapshots 列並更新 devices 釋放磁碟、只留最近 N 版
執行清理不可還原（本產品不提供回收桶）。刪掉的舊版 config 無法再 Diff／下載。 生產環境請先 dry-run，並確認 N 符合合規保留要求。清理不會動到「每設備最新 N 份」以外的邏輯；也不會刪除非 snapshot 結構的其他 store 檔案。

使用 — CDP/LLDP 鄰居

  • 選擇設備後，自最新快照讀取 cdp_neighbors.txt / lldp_neighbors.txt 解析鄰居表。
  • Cisco 且 CDP 有資料時，介面預設不與 LLDP 重複合併（依解析邏輯）。
  • 版面：設備表在上、鄰居結果在下（與 Interface Map 相同互動模式）。

使用 — Device Interface Map

  • 合併 config.txt 介面區段與 interfaces.txt 狀態欄位。
  • 支援 Cisco / Huawei / Fortinet 等已實作廠牌；表內容依設備而異。
  • 選設備後於下方顯示埠位表；設備列表同樣為固定高度捲動。

使用 — 排程備份

側欄「排程備份」路徑 /schedules。以日為週期對已納入清單的設備執行真實備份（與批次備份共用 runner／NetDriver Agent），結果寫入 store/ 快照與 index.db。

權限

角色排程頁
admin／operator建立、編輯、啟停、立即備份、刪除；可檢視執行歷史
viewer側欄可進入排程頁（唯讀）：檢視排程列表、設備清單、執行歷史；無法上傳 CSV 或觸發備份

viewer 側欄不顯示「批次備份」，其餘頁面（總表、鄰居、介面、排程）可讀。

資料存放

  • SQLite：store/schedules.db（排程定義、加密 SSH 憑證、schedule_runs 執行記錄）。
  • 加密主金鑰：首次儲存 live 憑證時自動建立 store/.secrets/fernet.key；亦可由環境變數 NCCM_SECRETS_KEY 覆寫。
  • 排程欄位：名稱、CSV 全文（僅含探測成功設備）、間隔（日）、SSH 帳密（加密）、啟用狀態、上次執行／結果、進行中 job_id。
  • CSV 格式與批次備份相同：至少 Site,IP,Vendor,Port 等（見「設備清單 CSV」）。

建立排程（admin／operator）

  1. 填名稱、間隔（日）（最小 1）、SSH 帳密（Enable 可選）。
  2. 選擇 CSV 檔案後按「上傳並連線測試」（僅檔案上傳，不支援貼上文字）。
  3. Portal 呼叫 NetDriver Agent POST /api/v1/probe 對每台設備做 TCP 連線測試。
  4. 預覽頁顯示每台 OK／FAIL；按「確認建立排程」後，僅 OK 設備寫入排程。

列表操作（admin／operator）

按鈕行為

執行歷史

排程頁下方「執行歷史」列出最近執行記錄（預設 30 筆），含排程名稱、開始／結束時間、狀態（running／done／failed／skipped）、觸發來源（manual／watcher）、成功台數與 job 摘要。所有角色皆可檢視。

背景自動觸發（watcher）

  • Portal 啟動時開 daemon 執行緒 nccm-schedule-watcher，約每 60 秒掃描一次。
  • 以 store/schedule.lock flock 避免多實例同時執行（single-instance 契約）。
  • 對每筆已啟用排程：若從未跑過，或距 last_run_at 已滿「間隔日」，則 execute_schedule 啟動真實備份 job。
  • 進行中 job 由 poll_running_jobs 輪詢完成後寫入 schedule_runs 與排程摘要。

store/

manifest.json 含 site、ip、port、hostname、vendor、model、run_id 等中繼資料。

資料 — 索引與重建

  • 設備主鍵：site::ip::port::hostname。
  • 每次成功備份會 upsert 設備與快照記錄。
  • 若曾手動刪除 index.db 或從他處還原 store/，請在 Web 按「重建索引」或執行 python -m nccm index rebuild。

維運 — Docker 更新方式

推薦標準更新（日常改動後）

git pull
docker compose build --no-cache
docker compose up -d

安全完整更新（升級版本或行為異常時建議）

docker compose down
git pull
docker compose build --no-cache
docker compose up -d --build

強制清除舊 Volume（Agent unhealthy、權限問題等）

docker compose down -v
git pull
docker compose build --no-cache
docker compose up -d --build

僅更新 Agent

docker compose build --no-cache netdriver-agent
docker compose up -d netdriver-agent

僅更新 Portal

docker compose build portal
docker compose up -d portal

更新後建議檢查

docker compose ps
docker compose logs -f netdriver-agent

重要提醒：

  • 重大更新前建議先備份 ./store 目錄。
  • 若本次更新涉及索引結構或 hostname 解析，更新後請在 Web 按「重建索引」。
  • --no-cache 可避免快取導致舊程式碼持續使用。

資安

  • 勿將 .env 提交版本庫；建議 chmod 600 .env。
  • Docker 使用 env_file，避免在 shell 歷史或 docker inspect 中留下明文密碼參數。
  • Web 密碼政策：長度 ≥12、不可為已知弱密碼、不可與帳號相同。
  • 設備 SSH 密碼僅存在於備份請求記憶體，不持久化。
  • 限制 :8000 對外暴露；僅維運網段可存取 OpenAPI。
  • CSV 含 WLC vendor 會被載入階段拒絕。

疑難排解

現象可能原因與處理
Portal 無法啟動檢查 .env 帳密是否符合政策；docker compose logs portal
側欄 Agent 紅燈Agent 未啟動、URL 錯誤、防火牆；curl http://localhost:8000/health

備份全部失敗Agent 容器無法 SSH 至設備；檢查路由／Port／帳密；必要時 host 網路
Forti 設定不完整提高 deploy/config/agent/agent.yml 中 Fortinet read_timeout
Huawei 備份 Unsupported mode: login舊版 Portal 對所有廠牌送 login；Huawei Agent 僅支援 enable。請使用目前 AAA-addon：payload 為 mode: enable（rebuild portal 若映像未更新）。
Cisco Nexus 備份失敗（auto classify 後）若 Detail 為 [config] …: ^ 但 CLI tail 已有設定與 hostname#，為 Agent 誤判 lone ^（請 rebuild netdriver-agent）。否則查權限／timeout；Nexus 用 show running-config，IOS 用 show running-config view full（login 模式、不用 enable）。
備份失敗但 docker log 無錯誤單台錯誤在 Portal SSE（含 [config] 與 CLI tail）與結果 Detail；若 FAILED — 後為空請 rebuild portal。Agent 可設 api.request_log_level: DEBUG 後查 docker compose logs netdriver-agent
庫存只有一台設備多設備同 Site+IP 須不同 Port 或 hostname；執行重建索引
設備列表 Hostname 為空白／unknown舊快照可能只從 version 抓到；新備份已優先從 running-config 解析。按「重建索引」可從現有 config.txt 回填。
鄰居／埠位空白該次備份未擷取到 CDP/LLDP 或 interfaces；重新備份並確認權限
建置 Agent 失敗確認可拉取 ghcr.io 基礎映像；代理或鏡像設定
