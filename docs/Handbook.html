NCCM v3 使用手冊

NetdriverBackup — 網路設備組態備份、庫存與排程。本頁可由 Portal 側欄「使用手冊」或 /help 開啟。

安裝與啟動

git clone https://github.com/moltmotl5-bot/NetdriveBackup.git
cd NetdriveBackup
cp .env.example .env
# 編輯 NCCM_ADMIN_PASS（至少 12 字元）
chmod 600 .env
mkdir -p store
docker compose up -d --build

瀏覽器開啟 http://localhost:8501/login，以 .env 帳密登入。首次登入須變更密碼。

側欄 Agent 狀態為 Online 表示 Portal 可連 Agent。請定期備份主機 ./store 目錄。

角色

角色能做什麼
admin全部功能 + 使用者、API Token、審計
operator備份、排程、索引、retention；無使用者管理
viewer唯讀：總表、鄰居、介面、排程檢視；無批次備份

批次備份

  1. 側欄「批次備份」→ 上傳或貼上 CSV。
  2. 輸入 SSH 帳密（Enable 密碼視廠牌需要）。
  3. 按開始；頁面以 SSE 顯示每台設備 log。

成功後快照寫入 store/<Site>/<IP>__<Hostname>/snapshots/，並更新 index.db。

設備總表與版控

列出已索引設備；Stack／HA 會展開為多列。可選兩版比對 Config Diff。

admin／operator 可下載 config、重建索引（掃描 store）、快照保留（dry-run 後執行刪除）。

排程備份

  1. 填名稱、間隔（日）、SSH 帳密，上傳 CSV 檔案。
  2. Agent 對每台做 TCP 探測；預覽頁確認後，僅 OK 設備納入排程。
  3. 可「立即備份」或啟用後由背景 watcher 依日週期執行。

憑證加密存於 store/schedules.db；執行記錄在排程頁「執行歷史」。viewer 僅能檢視，不能建立或觸發。

CSV 格式

必填欄位：Site,IP,Vendor,Port

Site,IP,Vendor,Port
hq,10.1.1.10,cisco,22
hq,10.1.1.20,fortinet,22

Vendor：cisco、huawei、fortinet（不支援 WLC）。可選 Model、Version 省略則自動辨識。

資料目錄 store/

store/
├── index.db           # 設備索引
├── schedules.db       # 排程與執行歷史
├── portal_auth.db     # Portal 帳號
├── audit/audit.db     # 審計
└── <Site>/<IP>__<Hostname>/snapshots/<時間>/
    ├── config.txt
    ├── manifest.json
    └── …

REST API

admin 於「API Token」建立 token，請求加標頭 X-API-Key。例：

curl -H "X-API-Key: nccm_…" http://localhost:8501/api/v1/inventory

詳細欄位見 Portal「API Token」頁內說明。

疑難排解

現象建議
Agent Offlinedocker compose ps、查 Agent log
備份全失敗Agent 容器能否 SSH 至設備 IP:Port
排程卡在 running等 job 完成；更新至最新 main；必要時查 schedules.db
Agent unhealthydocker compose down -v 後重建
