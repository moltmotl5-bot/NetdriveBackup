NetdriverBackup NCCM v3 使用手冊

企業網路設備組態備份、庫存與埠位檢視 — 安裝、設定與日常操作完整說明。
原始碼：github.com/moltmotl5-bot/NetdriveBackup

產品簡介

NCCM v3（Network Configuration Change Management）以 FastAPI Web 入口搭配 NetDriver Agent，透過 SSH 對網路設備執行 CLI，擷取組態與狀態並寫入本機 store/ 目錄與 SQLite 索引。

  • 支援廠牌：Cisco（IOS / NX-OS）、Fortinet FortiGate、Huawei CE（v3 路線圖不含 WLC）。
  • 四項 Web 功能：批次備份、設備總表與版控、CDP/LLDP 鄰居、Device Interface Map。
  • 正式部署：專案根目錄 docker-compose.yml（Portal + Agent）。舊版 Streamlit 已移至 legacy/。

環境需求

項目建議
Docker24 或以上，內建 Compose v2（docker compose）
作業系統Linux 或 macOS 部署主機
網路Agent 容器必須能 SSH 至 CSV 所列管理 IP／Port
瀏覽器Chromium、Safari、Firefox 等現代瀏覽器
磁碟依設備數與保留快照數成長；store/ 建議獨立掛載

安裝 — Docker Compose（建議）

  1. 取得專案原始碼並進入目錄： 
    git clone https://github.com/moltmotl5-bot/NetdriveBackup.git
cd NetdriveBackup
  2. 建立 Portal 登入設定： 
    cp .env.example .env
# 編輯 .env：設定 NCCM_ADMIN_USER、NCCM_ADMIN_PASS（至少 12 字元，不可為範例弱密碼）
chmod 600 .env
  3. 建立備份目錄並啟動服務： 
    mkdir -p store
docker compose up -d --build
    首次建置會編譯 netdrive-agent/ 與 Portal 映像，需能存取基礎映像 registry（含 ghcr.io）。
  4. 若 Agent 狀態為 unhealthy（常見於舊版 agent-logs volume）：docker compose down -v 後再 docker compose up -d --build。
  5. 開啟 Web：http://localhost:8501/login（埠可經 NCCM_PORT 調整）。
URL用途
http://<host>:8501/loginNCCM Web（需登入）
http://<host>:8501/healthPortal 健康檢查（JSON，含 Agent 是否可連）
http://<host>:8000/docsNetDriver OpenAPI（除錯；生產環境建議限制存取）
http://<host>:8000/healthAgent 健康檢查
停止服務：docker compose down。僅重啟 Portal：docker compose restart portal。

安裝 — 本機開發（非 Docker）

適用於修改 nccm/ 或 web/ 程式。Agent 仍需另行啟動（Docker 或本機 uv）。

python3 -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements-v3.txt
cp .env.example .env        # 設定帳密；本機時可設 NCCM_NETDRIVER_URL=http://127.0.0.1:8000
export PYTHONPATH=.

# 終端 A：啟動 Agent（見 netdrive-agent 文件或 docker compose up netdriver-agent）
# 終端 B：
python -m uvicorn web.main:app --reload --host 0.0.0.0 --port 8501

自檢腳本（ad-hoc，非正式測試套件）：

python3 scripts/hermes-verify-nccm-v3.py
python3 scripts/hermes-verify-neighbors-detail.py
python3 scripts/run-hermes-verify-inventory-config-download.py
python3 scripts/smoke-web-nccm-v3.py   # 需 Portal 已啟動

安裝後檢查

docker compose ps
docker compose logs -f portal --tail 50
docker compose logs -f netdriver-agent --tail 50
curl -s http://localhost:8501/health | python3 -m json.tool
curl -s http://localhost:8000/health

登入 Web 後，左側應顯示 Agent 已連線（綠色）。若為紅色，見 疑難排解。

設定 — Portal 環境變數

主要透過專案根目錄 .env 提供（Compose 使用 env_file）。

變數必填說明Docker Compose 預設
NCCM_ADMIN_USER是Web 登入帳號來自 .env
NCCM_ADMIN_PASS是Web 登入密碼，≥12 字元；不可等於帳號或內建弱密碼清單來自 .env
NCCM_NETDRIVER_URL建議Portal 呼叫 Agent 的 base URLhttp://netdriver-agent:8000
NCCM_STORE_DIR否備份根目錄（容器內路徑）/data/store，掛載 ./store
NCCM_APP_ROOT否應用根目錄/app
NCCM_SESSION_SECRET否Session cookie 簽章；多副本／重啟後要固定請設定未設則每次啟動隨機
API_KEY是（若使用 REST API）/api/v1 請求標頭 X-API-Key；未設時庫存 API 回 500來自 .env（見 .env.example）
NCCM_PORT否對外映射 Web 埠8501
NETDRIVER_AGENT_PORT否對外映射 Agent API 埠8000
NCCM_AUDIT_LOG否登入稽核 log 檔路徑預設 nccm_auth.log
設備 SSH 帳密僅在 Web「批次備份」頁輸入，不寫入 .env、映像或 store/。

設定 — REST API（API_KEY）

路徑前綴 /api/v1。與 Web Session 登入分開：程式整合請在 .env 設定 API_KEY，請求時帶 X-API-Key 標頭。

  • GET /api/v1/health — 無需金鑰。
  • GET /api/v1/inventory — 需有效 X-API-Key；伺服器未設定 API_KEY 時回 500（fail-closed）。
export API_KEY="your-long-random-secret"
curl -sS -H "X-API-Key: $API_KEY" "http://localhost:8501/api/v1/inventory?limit=20"

完整說明見專案 README.md「API 介面」章節。

設定 — NetDriver Agent

正式環境設定檔：deploy/config/agent/agent.yml，Compose 以唯讀方式掛載至容器 /app/config/agent/。

主要區段

區段用途
api.request_log_levelINFO 或 DEBUG（HTTP 請求記錄詳細度）
logging.levelAgent log 層級：INFO / DEBUG / TRACE
logging.log_file容器內路徑，預設 logs/agent.log；日誌請以 docker compose logs netdriver-agent 為主（勿掛載 root-owned volume 至 /app/logs）
session.ssh登入／連線逾時、keepalive、終端機寬高（大設定檔建議寬 1000、高 100）
session.profiles依 global → vendor/model → IP 覆寫 read_timeout、session 過期時間

廠牌 read_timeout（範例，已內建於 deploy 設定）

廠牌路徑（YAML）建議 read_timeout（秒）
Cisco Nexusvendor.cisco.nexus.base120
Fortinet FortiGatevendor.fortinet.fortigate.base180（大型 show full-configuration）
Huawei CEvendor.huawei.ce.base120
其他profiles.global30

修改 YAML 後：docker compose restart netdriver-agent。

環境變數 NETDRIVER_AGENT_CONFIG（Compose 已設）指向 /app/config/agent/agent.yml。

設定 — Compose 與網路

自訂對外埠

# .env
NCCM_PORT=9443
NETDRIVER_AGENT_PORT=18000

Agent 使用 host 網路

當設備管理網僅主機可達時，編輯 docker-compose.yml 的 netdriver-agent：

netdriver-agent:
  network_mode: host
  # 移除 ports: 與 networks: 設定

並將 portal 的 NCCM_NETDRIVER_URL 改為 http://127.0.0.1:8000（若 Portal 仍在 bridge，需確保能連到 host 上的 8000；常見做法是 Portal 也改 host 網路）。

僅對外暴露 Portal

生產環境可移除 netdriver-agent 的 ports 映射，僅讓 Portal 在內部網路呼叫 Agent。

使用 — 登入與側欄

  1. 開啟 /login，輸入 .env 的 NCCM_ADMIN_USER / NCCM_ADMIN_PASS。
  2. 登入成功後導向「批次備份」；左側導覽四頁：批次備份、設備總表與版控、CDP/LLDP 鄰居、Interface Map。
  3. 側欄顯示 store 路徑、Agent URL、Agent 連線狀態。
  4. /logout 可登出。

使用 — 批次備份

  1. 準備 CSV（上傳檔案或貼上文字）。
  2. 輸入設備 SSH 使用者名稱、密碼；Cisco 類設備若有 enable 密碼可填第三欄。
  3. 按開始備份；工作於背景執行，頁面以 SSE 串流顯示每台設備進度與錯誤。
  4. 成功後快照寫入 store/ 並更新 index.db。

每台設備大致流程

  • 依 Vendor 連線 NetDriver（必要時先以 version 指令自動辨識 Model/Version 後重連）。
  • 擷取 running config、version、CDP/LLDP（依廠牌）、interfaces 等。
  • FortiGate 使用 show full-configuration；Huawei 使用 display current-configuration 等。
同一表單一組 SSH 帳密套用到 CSV 全部列。若各廠牌帳密不同，請分次備份或拆成多份 CSV。

使用 — 設備總表與版控

  • 列表來自 index.db，可依 Site、Vendor（下拉，選項為已備份設備之廠牌）、關鍵字（IP / Hostname / Model / SN）篩選；鄰居與 Interface Map 頁同樣使用 Vendor 下拉。
  • 設備表約顯示 10 列高度，其餘於表內捲動；點列或「檢視」開啟下方詳情。
  • 詳情含型號、OS 版本、序號、快照時間列表；可預覽該次 config.txt（Running-Config），並以下載取得完整檔案（GET /inventory/download/config，需登入）。
  • Hostname 優先從 running-config 擷取；若舊快照顯示 unknown，可按「重建索引」從 config.txt 回填。
  • Cisco Stack / FortiGate HA：總表、鄰居、Interface Map 的設備列表皆每台實體機一行（Stack#、Role Primary/Secondary/Member、各自 Model/Serial）。Cisco 成員 hostname 為 管理名 · SW#；Forti 為 HA 解析之主機名。錨點列標 · 組態；虛擬 IP 只備一份 config。Cisco 備份含 show switch（stack_info.txt，非 Nexus）。異常時重新備份後按「重建索引」。
  • 重建索引：掃描整個 store/ 重建 index.db（手動還原資料或修復索引時使用）。

使用 — CDP/LLDP 鄰居

  • 選擇設備後，自最新快照讀取 cdp_neighbors.txt / lldp_neighbors.txt 解析鄰居表。
  • Cisco 且 CDP 有資料時，介面預設不與 LLDP 重複合併（依解析邏輯）。
  • 版面：設備表在上、鄰居結果在下（與 Interface Map 相同互動模式）。

使用 — Device Interface Map

  • 合併 config.txt 介面區段與 interfaces.txt 狀態欄位。
  • 支援 Cisco / Huawei / Fortinet 等已實作廠牌；表內容依設備而異。
  • 選設備後於下方顯示埠位表；設備列表同樣為固定高度捲動。

使用 — 命令列 CLI

無 Web 時可從專案根目錄執行（需 Agent 可連、PYTHONPATH=.）：

備份

export PYTHONPATH=.
export NCCM_NETDRIVER_URL=http://127.0.0.1:8000
python -m nccm backup \
  --csv DEMO-v3.csv \
  --user admin \
  --password '***' \
  --enable-password ''

重建索引

python -m nccm index rebuild

結束碼：備份若任一台失敗回傳 2。

資料 — 備份儲存結構

store/
├── index.db
└── <Site>/
    └── <IP>__<Hostname>/
        └── snapshots/
            └── <UTC-timestamp>/
                ├── manifest.json
                ├── config.txt
                ├── version_info.txt
                ├── cdp_neighbors.txt
                ├── lldp_neighbors.txt
                └── interfaces.txt

manifest.json 含 site、ip、port、hostname、vendor、model、run_id 等中繼資料。

資料 — 索引與重建

  • 設備主鍵：site::ip::port::hostname。
  • 每次成功備份會 upsert 設備與快照記錄。
  • 若曾手動刪除 index.db 或從他處還原 store/，請在 Web 按「重建索引」或執行 python -m nccm index rebuild。

維運 — Docker 更新方式

推薦標準更新（日常改動後）

git pull
docker compose build --no-cache
docker compose up -d

安全完整更新（升級版本或行為異常時建議）

docker compose down
git pull
docker compose build --no-cache
docker compose up -d --build

強制清除舊 Volume（Agent unhealthy、權限問題等）

docker compose down -v
git pull
docker compose build --no-cache
docker compose up -d --build

僅更新 Agent

docker compose build --no-cache netdriver-agent
docker compose up -d netdriver-agent

僅更新 Portal

docker compose build portal
docker compose up -d portal

更新後建議檢查

docker compose ps
docker compose logs -f netdriver-agent

重要提醒：

  • 重大更新前建議先備份 ./store 目錄。
  • 若本次更新涉及索引結構或 hostname 解析，更新後請在 Web 按「重建索引」。
  • --no-cache 可避免快取導致舊程式碼持續使用。

維運 — 日誌

來源查看方式
Portal 容器 stdoutdocker compose logs -f portal
Agent 容器 stdoutdocker compose logs -f netdriver-agent（建議）
Agent 容器內檔案 log可選 docker compose exec netdriver-agent tail -f /app/logs/agent.log（未掛載 logs volume 時）
Web 登入稽核NCCM_AUDIT_LOG（預設 nccm_auth.log）

資安

  • 勿將 .env 提交版本庫；建議 chmod 600 .env。
  • Docker 使用 env_file，避免在 shell 歷史或 docker inspect 中留下明文密碼參數。
  • Web 密碼政策：長度 ≥12、不可為已知弱密碼、不可與帳號相同。
  • 設備 SSH 密碼僅存在於備份請求記憶體，不持久化。
  • 限制 :8000 對外暴露；僅維運網段可存取 OpenAPI。
  • CSV 含 WLC vendor 會被載入階段拒絕。

疑難排解

現象可能原因與處理
Portal 無法啟動檢查 .env 帳密是否符合政策；docker compose logs portal
側欄 Agent 紅燈Agent 未啟動、URL 錯誤、防火牆；curl http://localhost:8000/health
Agent unhealthy多為舊 agent-logs volume 導致無法寫 log：docker compose down -v 後重建；查 docker compose logs netdriver-agent
備份全部失敗Agent 容器無法 SSH 至設備；檢查路由／Port／帳密；必要時 host 網路
Forti 設定不完整提高 deploy/config/agent/agent.yml 中 Fortinet read_timeout
Cisco Nexus 備份失敗（auto classify 後）若 Detail 為 [config] …: ^ 但 CLI tail 已有設定與 hostname#，為 Agent 誤判 lone ^（請 rebuild netdriver-agent）。否則查權限／timeout；使用 show running-config。
備份失敗但 docker log 無錯誤單台錯誤在 Portal SSE（含 [config] 與 CLI tail）與結果 Detail；若 FAILED — 後為空請 rebuild portal。Agent 可設 api.request_log_level: DEBUG 後查 docker compose logs netdriver-agent
庫存只有一台設備多設備同 Site+IP 須不同 Port 或 hostname；執行重建索引
設備列表 Hostname 為空白／unknown舊快照可能只從 version 抓到；新備份已優先從 running-config 解析。按「重建索引」可從現有 config.txt 回填。
鄰居／埠位空白該次備份未擷取到 CDP/LLDP 或 interfaces；重新備份並確認權限
建置 Agent 失敗確認可拉取 ghcr.io 基礎映像；代理或鏡像設定
