🛡️ AutoSwitchBackup2 Docker 映像部署

依 Streamlit 官方 Docker 教學 改寫，並加入本專案（NCCM 登入、output/ 備份目錄、Netmiko）所需設定。

前置需求

  • 已安裝 Docker Desktop 或 Docker Engine（macOS / Linux / Windows WSL2）
  • 專案路徑：/Users/claw/workspace/AutoSwitchBackup/AutoSwitchBackup2（或你實際 clone 的位置）
  • 容器需能連到交換機 SSH（批次備份時）；僅瀏覽既有 output/ 則只需掛載資料夾

步驟一：在專案根目錄新增 Dockerfile

  1. 進入專案目錄： 
    cd /Users/claw/workspace/AutoSwitchBackup/AutoSwitchBackup2
  2. 建立 Dockerfile（內容如下，與 Streamlit 文件建議一致）： 
    # syntax=docker/dockerfile:1
FROM python:3.12-slim

WORKDIR /app

# 系統相依（Netmiko / SSH 建議）
RUN apt-get update && apt-get install -y --no-install-recommends \
    openssh-client \
    && rm -rf /var/lib/apt/lists/*

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY app.py cdp_lldp_neighbors.py ./

# Streamlit 預設埠
EXPOSE 8501

HEALTHCHECK CMD curl --fail http://localhost:8501/_stcore/health || exit 1

ENTRYPOINT ["streamlit", "run", "app.py", \
  "--server.port=8501", \
  "--server.address=0.0.0.0", \
  "--server.headless=true", \
  "--browser.gatherUsageStats=false"]

    若映像建置時 HEALTHCHECK 失敗，可在 slim 映像加上 curl：apt-get install -y curl，或暫時刪除 HEALTHCHECK 那一行。

步驟二：新增 .dockerignore

  1. 避免把 venv、大量備份與機密打進映像： 
    venv/
__pycache__/
*.pyc
.env
output/
.git/
.DS_Store
*.md
!README.md

步驟三：建置 Docker 映像

  1. 在含 Dockerfile 的目錄執行： 
    docker build -t autoswitchbackup2-nccm:latest .
  2. 確認映像存在： 
    docker images autoswitchbackup2-nccm

步驟四：以容器執行（建議掛載 output 與環境變數）

  1. 登入帳密請用環境變數（勿把真實 .env bake 進映像）：
    變數說明
    NCCM_ADMIN_USERStreamlit 入口登入帳號（對應 app.py）
    NCCM_ADMIN_PASSStreamlit 入口登入密碼
  2. 第一次執行（本機 output 掛載進容器，備份會寫在 host 上）： 
    chmod 600 .env
docker run -d \
  --name nccm-portal \
  -p 8501:8501 \
  --env-file .env \
  -v "$(pwd)/output:/app/output" \
  autoswitchbackup2-nccm:latest
  3. 瀏覽器開啟：http://localhost:8501
  4. 查看日誌： 
    docker logs -f nccm-portal
備份目錄結構（重要）
程式預期：output/<Site>/<IP>_<Hostname>/<時間戳>/config.txt 等。
若只有 output/<IP>_<Hostname>/ 少一層 Site，設備總表的 Vendor／Hostname／序號會解析錯誤。請與 devices.csv 的 Site 欄一致。

步驟五（選用）：docker-compose.yml

  1. 方便重啟與管理環境變數： 
    services:
  nccm:
    build: .
    image: autoswitchbackup2-nccm:latest
    container_name: nccm-portal
    ports:
      - "8501:8501"
    environment:
      NCCM_ADMIN_USER: admin
      NCCM_ADMIN_PASS: ${NCCM_ADMIN_PASS}
    volumes:
      - ./output:/app/output
    restart: unless-stopped
  2. 在專案目錄放 .env（僅給 compose 讀取，例如 NCCM_ADMIN_PASS=...），然後： 
    docker compose up -d --build

步驟六：驗證與維運

  1. 登入後測試三個側欄：批次備份、設備總表、CDP/LLDP 鄰居。
  2. 停止／刪除容器： 
    docker stop nccm-portal
docker rm nccm-portal
  3. 更新程式後重新建置： 
    docker build -t autoswitchbackup2-nccm:latest .
docker stop nccm-portal && docker rm nccm-portal
# 再執行步驟四的 docker run 或 docker compose up -d --build

與 Streamlit 官方文件對照

  • 基底映像：python:3.x-slim + pip install -r requirements.txt
  • 啟動：streamlit run app.py --server.address=0.0.0.0 --server.port=8501
  • 對外映射：-p 8501:8501
  • 健康檢查：官方範例使用 /_stcore/health（見 Deploy Streamlit using Docker）

本專案額外項目：掛載 output、設定 NCCM_* 環境變數、容器網路需能 SSH 至設備網段（Netmiko）。
