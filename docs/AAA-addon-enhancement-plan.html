AAA-addon 系統增強計畫

分支 AAA-addon 已完成 AAA（使用者／Token）、多廠牌備份與 Huawei 序號／LLDP／stack 基礎。 本文件定義後續增強路徑：每個開發部件列出外部資料／實機備份前置事項，並附可直接貼給 Agent 的啟動 Prompt。

基準分支 AAA-addon 對齊 HEAD 約 56e04fa 起（Huawei LLDP UI） 語言 繁體中文交付 更新 repo = 程式 + README + Handbook + commit + push

一、現況與目標

已具備

  • AAA：portal_auth.db、admin／viewer、強制改密、API Token、側欄 logout
  • 備份：Cisco／Fortinet／Huawei 分廠牌 mode 與指令；不支援 WLC
  • 庫存：Cisco Stack、Forti HA、Huawei manufacture 序號與 ≥2 slot stack 展開
  • 鄰居：Huawei display lldp neighbor brief（含實機 Neighbor Dev 欄位）
  • 驗證：多支 scripts/run-hermes-verify-*.py（尚無正式 pytest／CI）

目標方向

從「可部署工具」→「可驗證、可排程、廠牌與實機一致、可營運」的 NCCM 平台。 不做：WLC、大框架重寫、破壞 Cisco／Fortinet 已綠路徑。

下列路徑建議統一放在倉庫外或 testdata/private/（gitignore）， 提交前必須去敏（密碼、社群／內網 IP 可改假值；hostname／serial 格式需保留真實形狀）。

三、開發慣例（所有階段共用）

  • 回覆／文件預設繁體中文；commit message 可用英文 conventional commits。
  • 禁止為修 Huawei 而改壞 Cisco／Fortinet 既有行為；變更後必跑對應 verify。
  • 驗證：優先 scripts/run-hermes-verify-*.py；P0 後改 pytest 並保留 ad-hoc 入口。
  • parser／index 變更後：部署 rebuild portal + Web「重建索引」。
  • 使用者說「update the repo」= 程式 + README + Handbook + commit + push。
  • 大功能可先 PLAN ONLY；確認後再 start building。

P0穩定與可驗證（約 1–2 週）

目標：回歸可機器判定；實機表頭進黃金樣本；Compose smoke 可一鍵跑。

P0.1 · pytest 最小套件

前置事項（外部資料）

  • 可選：現有 ad-hoc 腳本內嵌字串即可開工；不強制實機檔。
  • 若有 HW-LLDP-AS／HW-MFG-SINGLE 去敏副本 → 一併納入 tests/fixtures/。
  • 確認本機／CI 可用 Python 3.11+ 與專案 venv（或 Compose 內 portal 映像）。

工作內容

  • 建立 tests/：parsers（lldp／manufacture／stack）、auth token scope、API inventory 契約。
  • 把 run-hermes-verify-huawei-*.py 等收斂為 pytest 用例，腳本改呼叫或保留薄包裝。
  • GitHub Action 或本地 make test／pytest -q。

驗收

  • pytest 綠；既有 ad-hoc 腳本仍 PASSED。
  • 文件註明正式測試入口與 ad-hoc 差異。

P0.2 · Parser 黃金樣本（含 Huawei 實機）

前置事項（外部資料）— 必備

  • 必備 HW-LLDP-AS：至少含表頭 Local Intf / Neighbor Dev / Neighbor Intf / Exptime(s) 與 MultiGE／40GE 列。
  • 必備 HW-MFG-SINGLE：Slot/Sub + 12 字元 Serial 表。
  • 建議 HW-MFG-STACK：≥2 serial 列。
  • 建議 SimuNet huawei_ce.yml 輸出與實機表頭對照表（文件一段即可）。
  • 回歸必備 CS-STACK 最小片段、FG-HA 最小片段（可從現有 verify 抽出）。

工作內容

  • testdata/huawei/（或 private）放置去敏 fixture；測試 assert 筆數與欄位映射。
  • SimuNet 若與實機欄位順序不同，parser 必須雙支援（已部分完成，需鎖測）。

驗收

實機 42 筆 LLDP 樣本固定筆數；單機 manufacture 不展開 stack；Cisco／Fortinet fixture 不回退。

P0.3 · 索引／API 契約 + Compose smoke

前置事項（外部資料）

  • Docker Compose 可起 portal + agent；lab 可選 SimuNet，不必真實 SSH。

  • 文件化 serial_summary、stack 展開、API JSON 欄位。
  • scripts/smoke-compose.sh：health、login 或 API inventory、可選重建索引。
